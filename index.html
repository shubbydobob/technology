CS 개념 정리

Windows .inf 파일과 DNS를 독립적인 노트로 정리했습니다. 핵심 정의, 흐름, 비교 표, 면접 포인트를 한 화면에서 빠르게 확인할 수 있습니다.

운영체제

.inf란 무엇인가?

.inf 파일은 Windows에서 장치 드라이버, 시스템 구성 요소, 설치 정보 등을 설명하는 설정 정보 파일입니다.

핵심 개념

항목설명
정식 의미Setup Information File
파일 형식텍스트 기반 설정 파일
주 사용처Windows 드라이버 설치, 장치 인식, 시스템 구성 요소 설치
실행 파일 여부직접 실행되는 프로그램이 아니라 Windows 설치 관리자나 Plug and Play 관리자가 해석합니다.
대표 위치C:\Windows\INF

설치 흐름

관련 파일 비교

파일역할
.inf설치 절차와 장치 매칭 정보를 설명합니다.
.sys커널 모드 드라이버 본체인 경우가 많습니다.
.cat드라이버 패키지의 디지털 서명 카탈로그입니다.

.exe별도 설치 프로그램 또는 유틸리티일 수 있습니다.

면접 포인트

  • .inf는 실행 파일인가?아닙니다. Windows가 해석하는 텍스트 기반 설치 정보 파일입니다.
  • 드라이버 설치에서 역할은?하드웨어 ID와 드라이버를 매칭하고 설치 절차를 정의합니다.
  • .inf만 있으면 드라이버가 동작하는가?아닙니다. 실제 드라이버 파일인 .sys 등이 함께 필요합니다.

네트워크

DNS란 무엇인가?

DNS는 사람이 읽기 쉬운 도메인 이름을 컴퓨터가 통신에 사용하는 IP 주소로 변환해주는 인터넷의 이름 해석 시스템입니다.

핵심 개념

항목설명
정식 명칭Domain Name System
주 역할도메인 이름을 IP 주소로 변환
비유인터넷의 전화번호부
기본 포트UDP 53, 필요 시 TCP 53
대표 레코드A, AAAA, CNAME, MX, NS, TXT
구조계층형 분산 데이터베이스

주요 DNS 서버 역할

구성 요소역할
Stub Resolver사용자의 OS나 브라우저 쪽에서 DNS 질의를 시작합니다.
Recursive Resolver사용자를 대신해 Root, TLD, 권한 DNS 서버에 질의합니다.
Root DNS Server최상위 DNS 서버로 TLD 서버 위치를 알려줍니다.
TLD DNS Server.com, .kr 같은 최상위 도메인의 담당 권한 서버를 알려줍니다.
Authoritative DNS Server특정 도메인의 실제 DNS 레코드를 보유하고 최종 응답을 제공합니다.

주요 DNS 레코드

레코드의미예시 사용처
A도메인을 IPv4 주소에 연결웹 서버 접속
AAAA도메인을 IPv6 주소에 연결IPv6 기반 접속
CNAME별칭 도메인을 실제 도메인에 연결www 별칭 구성
MX메일 서버 지정이메일 수신 서버 설정
NS권한 DNS 서버 지정도메인 위임
TXT텍스트 정보 저장SPF, DKIM, 소유권 검증
PTRIP 주소를 도메인 이름으로 역조회Reverse DNS

캐시와 TTL

개념설명
DNS Cache이전 조회 결과를 저장해 재조회 속도를 높입니다.
TTL캐시된 DNS 응답이 유효한 시간입니다.
TTL이 짧을 때변경 반영은 빠르지만 DNS 질의가 많아집니다.
TTL이 길 때성능은 좋지만 변경 전파가 느립니다.

면접 포인트

  • DNS는 왜 필요한가?사람이 읽는 도메인을 네트워크가 사용하는 IP 주소로 변환하기 위해 필요합니다.
  • Recursive Resolver와 Authoritative DNS의 차이는?Recursive Resolver는 대신 찾아주는 서버이고, Authoritative DNS는 실제 정답 레코드를 가진 서버입니다.
  • DNS는 항상 Root 서버까지 질의하는가?아닙니다. 캐시에 있으면 Root까지 가지 않고 바로 응답합니다.
  • DNS가 UDP를 쓰는 이유는?대부분의 질의가 작고 빠른 응답이 중요하기 때문입니다.
