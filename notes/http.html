HTTP란 무엇인가?

HTTP는 HyperText Transfer Protocol의 약자로, 브라우저와 서버가 데이터를 주고받는 규칙입니다. 요청(Request)과 응답(Response) 구조를 가지며 기본적으로 상태를 저장하지 않는 stateless 특성을 가집니다.

간단한 예시

GET /api/users/1 HTTP/1.1
Host: example.com

HTTP/1.1 200 OK
Content-Type: application/json
{"id":1,"name":"kim"}

핵심 구성

구성설명
MethodGET, POST, PUT, DELETE처럼 요청 목적을 나타냅니다.
URL요청할 자원의 위치입니다.
Header인증, 콘텐츠 타입, 쿠키 같은 부가 정보입니다.
BodyPOST/PUT 요청에서 서버로 보내는 데이터입니다.
Status Code200, 404, 500처럼 응답 결과를 나타냅니다.

기술 포인트

  • HTTP가 stateless라는 뜻은?서버가 각 요청을 독립적으로 처리한다는 뜻입니다. 로그인 유지에는 쿠키, 세션, 토큰 같은 별도 장치가 필요합니다.
  • GET과 POST 차이는?GET은 주로 조회, POST는 생성이나 처리 요청에 사용합니다. GET은 안전성과 멱등성을 기대하고 POST는 그렇지 않을 수 있습니다.
