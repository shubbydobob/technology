Spring MVC

ModelAttribute란 무엇인가?

ModelAttribute는 요청 파라미터를 객체에 바인딩하거나 View에 전달할 Model 데이터를 준비하는 Spring MVC 기능입니다.

Spring MVC

ModelAttribute란 무엇인가?

@ModelAttribute는 요청 파라미터를 Java 객체에 바인딩할 때 사용합니다. 또한 컨트롤러 메서드 실행 전에 공통 Model 데이터를 미리 추가하는 용도로도 사용할 수 있습니다.

간단한 예시

검색 화면에서 ?keyword=spring&page=1 같은 요청이 들어오면 Spring은 요청 파라미터를 SearchCondition 객체의 필드에 바인딩할 수 있습니다.

public class SearchCondition {
    private String keyword;
    private int page;
}

@GetMapping("/posts")
public String posts(@ModelAttribute SearchCondition condition) {
    // condition.keyword = "spring"
    // condition.page = 1
    return "posts/list";
}

주요 역할

역할설명
객체 바인딩쿼리 파라미터나 form 데이터를 객체 필드에 매핑합니다.
Model 추가View 렌더링에 필요한 데이터를 Model에 담습니다.
검증 연계@Valid, BindingResult와 함께 form 검증에 자주 사용됩니다.

RequestBody와 차이

구분@ModelAttribute@RequestBody
주 입력query string, form dataHTTP body의 JSON/XML 등
사용 맥락HTML form, 검색 조건, 화면 기반 요청REST API JSON 요청
변환 방식데이터 바인더가 필드에 매핑HttpMessageConverter가 본문을 객체로 변환

면접 포인트

  • @ModelAttribute는 언제 쓰는가?form 데이터나 query parameter를 객체로 받을 때 자주 사용합니다.
  • @RequestBody와 핵심 차이는?ModelAttribute는 파라미터/form 바인딩 중심이고, RequestBody는 요청 본문 변환 중심입니다.
  • 어노테이션을 생략해도 되는가?복합 객체 파라미터는 Spring MVC에서 ModelAttribute처럼 처리되는 경우가 많지만, 의도를 명확히 하려면 붙이는 편이 좋습니다.
