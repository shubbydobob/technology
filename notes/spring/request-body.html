Spring MVC

@RequestBody란 무엇인가?

@RequestBody는 HTTP 요청 본문에 담긴 JSON/XML 등의 데이터를 Java 객체로 변환해 Controller 메서드 파라미터에 바인딩하는 Spring MVC 어노테이션입니다.

핵심 개념

항목설명
사용 위치Controller 메서드 파라미터
주요 역할HTTP Body를 Java 객체로 역직렬화
대표 입력JSON 요청 본문
내부 처리HttpMessageConverter가 변환 수행
자주 함께 쓰는 어노테이션@Valid, @PostMapping, @PutMapping, @PatchMapping

요청 처리 흐름

sequenceDiagram participant Client participant Controller participant Converter as HttpMessageConverter participant Service Client->>Controller: POST /users JSON Body Controller->>Converter: Body를 Java 객체로 변환 요청 Converter-->>Controller: CreateUserRequest 객체 Controller->>Service: 비즈니스 로직 호출 Service-->>Controller: 결과 반환 Controller-->>Client: Response 반환

예시

@PostMapping("/users")
public ResponseEntity<UserResponse> createUser(
    @RequestBody CreateUserRequest request
) {
    UserResponse response = userService.create(request);
    return ResponseEntity.ok(response);
}

@RequestBody와 @RequestParam 차이

구분@RequestBody@RequestParam
읽는 위치HTTP BodyQuery String 또는 Form Data
주 사용처JSON API 요청검색 조건, 단순 파라미터
예시{ "name": "kim" }/users?name=kim
변환 방식HttpMessageConverterTypeConverter/Formatter

면접 포인트

  • @RequestBody는 무엇인가?HTTP 요청 본문을 Java 객체로 바인딩하는 어노테이션입니다.
  • JSON을 객체로 바꾸는 주체는?HttpMessageConverter, 보통 MappingJackson2HttpMessageConverter가 처리합니다.
  • GET 요청에서도 쓰는가?기술적으로 가능할 수 있지만 일반적으로 GET은 Body를 사용하지 않고 Query Parameter를 씁니다.
  • 유효성 검증은?DTO에 검증 어노테이션을 붙이고 파라미터에 @Valid를 함께 사용합니다.
