Spring MVC

RestController와 Controller 차이

@Controller는 주로 View를 반환하는 MVC Controller이고, @RestController는 JSON/XML 같은 데이터를 HTTP Body로 바로 반환하는 REST API Controller입니다.

핵심 차이

구분@Controller@RestController
주 사용 목적화면(View) 반환데이터(JSON/XML) 반환
반환값 해석View 이름으로 해석될 수 있음HTTP Response Body로 작성됨
@ResponseBody 필요 여부JSON 반환 시 필요기본 포함
주 사용 위치전통적인 MVC 웹 페이지REST API 서버
예시 반환"user/list"{ "id": 1, "name": "kim" }

@RestController의 의미

@RestController는 내부적으로 @Controller와 @ResponseBody를 합친 어노테이션입니다. 따라서 클래스의 모든 핸들러 메서드에 @ResponseBody가 적용된 것처럼 동작합니다.

요청 처리 흐름 비교

flowchart TD A[Client Request] --> B{Controller 종류} B -->|@Controller| C[Controller Method] C --> D[View Name 반환] D --> E[ViewResolver] E --> F[HTML View 응답] B -->|@RestController| G[Controller Method] G --> H[객체 반환] H --> I[HttpMessageConverter] I --> J[JSON/XML Response Body]

코드 비교

@Controller
public class PageController {
    @GetMapping("/users")
    public String usersPage(Model model) {
        model.addAttribute("users", userService.findAll());
        return "users/list";
    }
}

@RestController
public class UserApiController {
    @GetMapping("/api/users")
    public List<UserResponse> users() {
        return userService.findAll();
    }
}

언제 무엇을 쓰는가?

상황선택
Thymeleaf/JSP 같은 서버 렌더링 화면을 반환한다@Controller
JSON 기반 REST API를 만든다@RestController
일부 메서드만 JSON을 반환한다@Controller + 메서드별 @ResponseBody
프론트엔드와 백엔드를 분리한 API 서버다대부분 @RestController

면접 포인트

  • @Controller와 @RestController의 차이는?@Controller는 View 반환 중심, @RestController는 Response Body 데이터 반환 중심입니다.
  • @RestController는 무엇의 조합인가?@Controller + @ResponseBody입니다.
  • @Controller에서 JSON을 반환하려면?메서드나 클래스에 @ResponseBody를 붙입니다.
  • 객체를 JSON으로 바꾸는 주체는?HttpMessageConverter, 보통 Jackson 기반 converter가 처리합니다.
