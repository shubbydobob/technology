Spring MVC

DTO란 무엇인가?

DTO(Data Transfer Object)는 계층 간 데이터를 전달하기 위한 객체입니다. Entity를 직접 노출하지 않고 요청/응답에 필요한 데이터만 담아 전달할 때 주로 사용합니다.

DTO를 사용하는 이유

이유설명
계층 분리API 요청/응답 모델과 DB Entity를 분리합니다.
보안Entity의 민감한 필드가 외부로 노출되는 것을 막습니다.
유효성 검증요청 DTO에 검증 어노테이션을 붙여 입력값을 검증할 수 있습니다.
API 명확성클라이언트에 필요한 데이터 형태를 명확히 정의합니다.

Entity와 DTO 차이

구분EntityDTO
목적DB 테이블과 매핑데이터 전달
위치Persistence 계층Controller/Service/API 계층
변경 이유DB 구조 변경API 스펙 변경
외부 노출지양가능

흐름

flowchart LR A[Client Request] --> B[Request DTO] B --> C[Controller] C --> D[Service] D --> E[Entity] E --> F[Repository] D --> G[Response DTO] G --> H[Client Response]

예시

public record CreateUserRequest(String name, String email) {}
public record UserResponse(Long id, String name, String email) {}

면접 포인트

  • DTO는 왜 쓰는가?계층 간 데이터 전달을 명확히 하고 Entity 직접 노출을 막기 위해서입니다.
  • Entity를 그대로 반환하면 안 되는 이유는?민감 정보 노출, 순환 참조, API와 DB 구조의 강한 결합 문제가 생길 수 있습니다.
  • Request DTO와 Response DTO를 나누는 이유는?요청과 응답의 데이터 요구사항이 다르기 때문입니다.
