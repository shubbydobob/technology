자주 쓰는 데이터를 더 빠른 저장소에 임시로 보관해 응답 속도와 처리량을 높이는 방식입니다.

백엔드

캐시란 무엇인가?

캐시(Cache)는 반복 조회되는 데이터를 DB보다 빠른 위치에 저장해 재사용하는 방식입니다. 응답 속도를 줄이고 DB 부하를 낮출 수 있지만 데이터 최신성 관리가 중요합니다.

Cache Aside 예시

1. 캐시 조회
2. 없으면 DB 조회
3. DB 결과를 캐시에 저장
4. 응답 반환

주요 개념

개념설명
Cache Hit캐시에 데이터가 있어 바로 응답하는 상황입니다.
Cache Miss캐시에 없어 원본 저장소를 조회하는 상황입니다.
TTL캐시 데이터가 유지되는 시간입니다.
Eviction캐시 공간 확보를 위해 데이터를 제거하는 정책입니다.

기술 포인트

  • 캐시의 가장 큰 위험은?원본 데이터와 캐시 데이터가 달라지는 불일치 문제입니다.
  • 언제 캐시를 쓰나?자주 조회되고 변경이 적으며 계산이나 DB 조회 비용이 큰 데이터에 적합합니다.
