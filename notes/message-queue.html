백엔드

메시지 큐란 무엇인가?

메시지 큐는 요청을 즉시 처리하지 않고 메시지 형태로 저장한 뒤, Consumer가 순차적으로 꺼내 처리하게 하는 구조입니다. 트래픽 완충, 비동기 처리, 서비스 분리에 유용합니다.

흐름

Producer -> Queue -> Consumer
주문 API -> 결제 요청 메시지 -> 결제 Worker

주요 개념

개념설명
Producer메시지를 발행하는 주체입니다.
Queue메시지를 임시 저장합니다.
Consumer메시지를 가져가 처리합니다.
Retry실패한 메시지를 다시 처리합니다.
DLQ반복 실패 메시지를 따로 보관하는 큐입니다.

기술 포인트

  • 메시지 큐를 쓰는 이유는?요청 처리와 실제 작업을 분리해 응답성을 높이고 장애 전파를 줄이기 위해 사용합니다.
  • 주의할 점은?중복 처리, 순서 보장, 재시도, 실패 메시지 처리를 설계해야 합니다.
