API는 서로 다른 프로그램이 정해진 방식으로 기능이나 데이터를 주고받기 위한 약속입니다.

백엔드

API란 무엇인가?

API는 Application Programming Interface의 약자입니다. 애플리케이션끼리 기능을 호출하거나 데이터를 주고받기 위해 정한 인터페이스입니다. 웹 개발에서는 보통 클라이언트가 서버에 요청하고 서버가 응답하는 HTTP API를 많이 말합니다.

간단한 예시

게시글 목록을 조회하는 화면이 있다고 가정합니다. 프론트엔드는 서버의 GET /api/posts API를 호출하고, 서버는 게시글 목록을 JSON으로 응답합니다.

GET /api/posts

HTTP/1.1 200 OK
Content-Type: application/json

[
  { "id": 1, "title": "첫 번째 글" },
  { "id": 2, "title": "두 번째 글" }
]

API 구성 요소

요소설명
Endpoint/api/posts처럼 요청을 보내는 주소입니다.
MethodGET, POST, PUT, DELETE처럼 요청의 목적을 나타냅니다.
Request클라이언트가 서버로 보내는 파라미터, 헤더, 본문입니다.
Response서버가 클라이언트에 돌려주는 상태 코드와 데이터입니다.
Contract요청/응답 형식, 인증 방식, 에러 규칙 같은 약속입니다.

왜 중요한가?

  • 역할 분리프론트엔드와 백엔드가 API 계약을 기준으로 독립적으로 개발할 수 있습니다.
  • 재사용웹, 앱, 외부 시스템이 같은 API를 재사용할 수 있습니다.
  • 통합결제, 지도, 로그인 같은 외부 서비스를 API로 연동할 수 있습니다.

면접 포인트

  • REST API란?자원을 URI로 표현하고 HTTP Method로 행위를 표현하는 API 설계 스타일입니다.
  • API 계약이 중요한 이유는?계약이 바뀌면 API를 사용하는 클라이언트 코드가 깨질 수 있기 때문입니다.
