백엔드

DI란 무엇인가?

DI(Dependency Injection)는 의존성 주입이라는 뜻입니다. 객체가 필요한 협력 객체를 직접 생성하지 않고 외부 컨테이너나 생성자를 통해 전달받습니다.

간단한 예시

class OrderService {
  private final PaymentClient paymentClient;
  OrderService(PaymentClient paymentClient) {
    this.paymentClient = paymentClient;
  }
}

장점

장점설명
결합도 감소구현체 교체가 쉬워집니다.
테스트 용이가짜 객체나 Mock을 주입할 수 있습니다.
관심사 분리객체 생성 책임과 사용 책임을 분리합니다.
확장성설정에 따라 다른 구현체를 사용할 수 있습니다.

기술 포인트

  • DI와 IoC의 관계는?IoC는 제어권을 외부로 넘기는 원칙이고, DI는 IoC를 구현하는 대표 방식입니다.
  • 생성자 주입을 선호하는 이유는?필수 의존성을 명확히 하고 불변성을 유지하며 테스트하기 쉽기 때문입니다.
