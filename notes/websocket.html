네트워크

WebSocket이란 무엇인가?

WebSocket은 클라이언트와 서버가 하나의 TCP 연결을 유지하면서 양방향으로 실시간 데이터를 주고받을 수 있게 해주는 통신 프로토콜입니다.

핵심 개념

항목설명
통신 방식양방향 Full-Duplex 통신
기반TCP
시작 방식HTTP Upgrade 핸드셰이크
대표 스킴ws://, wss://
사용처채팅, 실시간 알림, 게임, 협업 편집, 실시간 대시보드

HTTP와 WebSocket 차이

구분HTTPWebSocket
통신 모델Request/Response양방향 메시징
연결요청마다 생성하거나 Keep-Alive연결을 계속 유지
서버 푸시기본적으로 어려움서버가 즉시 전송 가능
적합한 경우일반 API, 페이지 요청실시간 통신

연결 흐름

sequenceDiagram participant C as Client participant S as Server C->>S: HTTP Request with Upgrade: websocket S-->>C: 101 Switching Protocols C<->>S: WebSocket 양방향 메시지 교환

주의할 점

항목설명
연결 관리많은 클라이언트의 장기 연결을 관리해야 합니다.
확장성서버 여러 대 구성 시 세션/브로커/로드밸런싱 전략이 필요합니다.
인증최초 연결 시 토큰 검증 등 인증 처리가 필요합니다.
장애 처리재연결, heartbeat, timeout 처리가 중요합니다.

면접 포인트

  • HTTP와 차이는?HTTP는 요청/응답 중심이고 WebSocket은 연결 유지 기반 양방향 메시징입니다.
  • 어떻게 시작되는가?HTTP Upgrade 요청 후 101 Switching Protocols 응답으로 전환됩니다.
