백엔드

Nginx란 무엇인가?

Nginx는 웹 서버이면서 리버스 프록시, 로드밸런서, 정적 파일 서버 역할을 수행할 수 있는 서버 소프트웨어입니다. 실무에서는 브라우저와 WAS 사이의 앞단에 두고 요청을 받아 정적 파일은 직접 응답하고, API 요청은 뒤쪽 애플리케이션 서버로 전달하는 구조로 많이 사용합니다.

간단한 예시

Vue, React 같은 프론트엔드 정적 파일은 Nginx가 직접 내려주고, /api로 시작하는 요청은 Spring Boot 같은 WAS로 전달할 수 있습니다.

Browser
  -> Nginx
    -> /index.html, /assets/* 는 Nginx가 직접 응답
    -> /api/* 는 WAS로 프록시

리버스 프록시란?

리버스 프록시는 클라이언트 앞에 있는 서버가 아니라, 서버 앞단에서 클라이언트 요청을 받아 내부 서버로 대신 전달하는 구조입니다. 사용자는 내부 WAS 주소를 직접 알 필요가 없고, Nginx는 요청 경로에 따라 적절한 서버로 요청을 보냅니다.

server {
  listen 80;

  location / {
    root /usr/share/nginx/html;
    index index.html;
  }

  location /api/ {
    proxy_pass http://127.0.0.1:8080/;
  }
}

WAS 앞에 Nginx를 두는 이유

  • 정적 파일 처리이미지, CSS, JS 같은 정적 파일은 Nginx가 효율적으로 처리하고 WAS는 비즈니스 로직에 집중할 수 있습니다.
  • 보안 경계내부 WAS를 외부에 직접 노출하지 않고 Nginx만 공개해 접근 경계를 나눌 수 있습니다.
  • HTTPS 처리인증서 관리와 TLS 처리를 Nginx에서 담당하면 애플리케이션 서버 설정을 단순화할 수 있습니다.
  • 트래픽 제어로드밸런싱, 요청 크기 제한, 타임아웃, 캐싱 같은 정책을 앞단에서 적용할 수 있습니다.

기술 포인트

  • Nginx는 WAS인가?아닙니다. Nginx는 주로 웹 서버와 리버스 프록시 역할을 하며, Java 비즈니스 로직을 직접 실행하는 WAS와는 역할이 다릅니다.
  • 리버스 프록시를 쓰는 이유는?내부 서버를 직접 노출하지 않고 요청 라우팅, HTTPS 처리, 로드밸런싱, 캐싱 같은 공통 처리를 앞단에서 담당하기 위해 사용합니다.
  • Nginx가 정적 파일 처리에 유리한 이유는?이벤트 기반 구조로 많은 동시 연결을 효율적으로 처리할 수 있고, 정적 파일 응답은 애플리케이션 로직 없이 빠르게 반환할 수 있기 때문입니다.
