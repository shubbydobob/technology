ref와 reactive 차이

ref는 단일 값과 객체를 .value로 감싸고, reactive는 객체 자체를 반응형 Proxy로 만드는 API입니다.

Vue3

ref와 reactive 차이

ref는 값을 담는 반응형 박스이고, reactive는 객체를 반응형 객체로 변환합니다. 원시값은 reactive로 직접 감쌀 수 없으므로 ref를 사용합니다.

예시

const count = ref(0)
count.value++

const user = reactive({ name: 'kim' })
user.name = 'lee'

비교

구분refreactive
대상원시값, 객체 모두 가능객체 중심
접근script에서 .value 필요속성에 직접 접근
권장단일 상태, composable 반환값관련 필드가 묶인 객체 상태

기술 포인트

  • ref는 왜 .value를 쓰나?원시값을 객체처럼 감싸 반응형 추적 대상이 되게 하기 위해서입니다.
  • reactive를 구조 분해하면?반응성이 끊길 수 있어 toRefs나 storeToRefs 같은 보조 API를 사용합니다.
