동시성 제어란 무엇인가?

여러 요청이나 스레드가 같은 데이터에 동시에 접근할 때 정합성을 지키기 위한 제어 방식입니다.

백엔드

동시성 제어란 무엇인가?

동시성 제어는 여러 작업이 같은 데이터를 동시에 읽고 수정할 때 잘못된 결과가 생기지 않도록 순서와 접근을 관리하는 기술입니다.

간단한 예시

재고 1개
사용자 A 구매 요청
사용자 B 구매 요청

제어가 없으면 둘 다 성공해 재고가 -1이 될 수 있음

대표 방식

방식설명
비관적 락충돌 가능성이 높다고 보고 먼저 락을 잡습니다.
낙관적 락충돌이 적다고 보고 버전 값으로 충돌을 감지합니다.
분산 락여러 서버 사이에서 하나의 작업만 진행되게 제어합니다.
트랜잭션 격리DB 격리 수준으로 동시 접근 영향을 제어합니다.

기술 포인트

  • 낙관적 락은 언제 쓰나?충돌이 드문 상황에서 성능을 유지하면서 충돌을 감지하고 재시도할 때 적합합니다.
  • 분산 락은 왜 필요한가?서버가 여러 대면 JVM 내부 락만으로는 전체 시스템의 동시 작업을 막을 수 없기 때문입니다.
