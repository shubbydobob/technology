터널링이란 무엇인가?

터널링은 한 네트워크 연결 안에 다른 트래픽을 감싸 전달하는 방식입니다.

네트워크

터널링이란 무엇인가?

터널링은 직접 접근하기 어려운 네트워크나 서비스에 중간 통로를 만들어 트래픽을 전달하는 방식입니다. 보통 SSH, VPN, 프록시, 전용 터널링 도구를 통해 구성합니다.

예시 흐름

로컬 PC에서 직접 접근할 수 없는 내부 DB가 있을 때, SSH로 접근 가능한 서버를 경유해 로컬 포트와 내부 DB 포트를 연결할 수 있습니다.

구간역할
localhost:3306개발자가 접속하는 로컬 포트
SSH 서버암호화된 중간 통로
internal-db:3306실제 목적지 DB

주요 사용 사례

  • 내부 DB 접속사설망에 있는 DB를 로컬 개발 환경에서 임시로 연결할 때 사용합니다.
  • 관리자 페이지 접근외부에 직접 공개하지 않은 관리 페이지에 안전하게 접근할 수 있습니다.
  • 로컬 서버 공개ngrok, Cloudflare Tunnel처럼 로컬 서비스를 임시 공개 URL로 노출할 수 있습니다.
  • VPN사설 네트워크에 들어온 것처럼 통신하기 위해 터널링을 사용합니다.

포트 포워딩과의 차이

구분포트 포워딩터널링
목적외부 요청을 내부 특정 포트로 전달별도 통로 안에 트래픽을 감싸 전달
설정 위치공유기, 방화벽, NAT 장비SSH, VPN, 프록시, 터널링 도구
보안 성격서비스 포트가 직접 노출될 수 있음암호화된 통로를 사용하는 경우가 많음
예시공인IP:80을 내부 8080으로 전달로컬 3306을 SSH 경유로 내부 DB에 연결

면접 포인트

  • 터널링의 핵심은?직접 전달하지 않고 다른 연결 안에 트래픽을 감싸 목적지까지 보내는 것입니다.
  • SSH 터널링은 언제 쓰는가?외부에서 직접 접근할 수 없는 내부 서비스에 안전하게 접근할 때 사용합니다.
  • 터널링은 항상 보안 기능인가?터널링 자체는 전달 방식이고, 보안성은 사용하는 프로토콜의 암호화와 인증에 따라 달라집니다.
