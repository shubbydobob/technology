Operating System

스레드란 무엇인가?

하나의 프로세스 안에서 실행 흐름을 나누어 작업을 동시에 처리할 수 있게 하는 실행 단위입니다.

운영체제

스레드란 무엇인가?

스레드(Thread)는 프로세스 안에서 실제로 코드를 실행하는 흐름입니다. 하나의 프로세스는 하나 이상의 스레드를 가질 수 있고, 여러 스레드는 같은 프로세스의 메모리 공간을 공유하면서 각자 독립적인 실행 흐름을 가집니다.

간단한 예시

웹 브라우저를 생각하면 화면을 그리는 작업, 네트워크 요청을 보내는 작업, 사용자 입력을 처리하는 작업이 동시에 진행됩니다. 이런 작업을 하나의 흐름으로만 처리하면 화면이 멈출 수 있으므로 여러 스레드나 비동기 구조를 사용해 작업을 나눕니다.

Process: Browser
  Thread 1: 화면 렌더링
  Thread 2: 네트워크 요청
  Thread 3: 사용자 입력 처리

프로세스와 스레드 차이

구분프로세스스레드
의미실행 중인 프로그램의 독립된 단위프로세스 안에서 실행되는 작업 흐름
메모리프로세스마다 독립된 메모리 공간을 가집니다.같은 프로세스의 Code, Data, Heap 영역을 공유합니다.
통신IPC 같은 별도 통신 방식이 필요합니다.공유 메모리를 통해 상대적으로 쉽게 데이터를 주고받습니다.
오버헤드생성, 전환 비용이 비교적 큽니다.생성, 전환 비용이 비교적 작습니다.
장애 영향한 프로세스 장애가 다른 프로세스에 직접 영향을 덜 줍니다.한 스레드 문제가 같은 프로세스 전체에 영향을 줄 수 있습니다.

스레드가 공유하는 것과 독립적으로 가지는 것

구분내용
공유Code, Data, Heap, 파일 디스크립터 같은 프로세스 자원
독립Stack, Register, Program Counter 같은 실행 흐름 정보

왜 사용하는가?

  • 동시 처리여러 작업을 동시에 진행해 응답성을 높일 수 있습니다. 예를 들어 서버는 여러 요청을 병렬로 처리할 수 있습니다.
  • 자원 공유같은 프로세스의 메모리를 공유하므로 데이터 전달 비용이 프로세스 간 통신보다 낮을 수 있습니다.
  • 응답성 개선시간이 오래 걸리는 작업을 별도 스레드에서 처리하면 메인 흐름이 멈추는 것을 줄일 수 있습니다.
  • CPU 활용멀티코어 환경에서 여러 스레드를 병렬 실행해 CPU 자원을 더 활용할 수 있습니다.

주의할 점

스레드는 메모리를 공유하기 때문에 여러 스레드가 같은 데이터를 동시에 수정하면 문제가 생길 수 있습니다. 대표적인 문제가 경쟁 상태(Race Condition)이며, 이를 막기 위해 락, synchronized, mutex, semaphore 같은 동기화 기법을 사용합니다.

문제 상황:
Thread A: count = count + 1
Thread B: count = count + 1

동시에 실행되면 기대한 값보다 적게 증가할 수 있음

기술 포인트

  • 스레드의 핵심은?프로세스 안에서 실행되는 흐름이며, 같은 프로세스의 메모리 자원을 공유한다는 점입니다. 그래서 빠른 협업이 가능하지만 동시성 문제도 함께 생깁니다.
  • 프로세스와 스레드의 가장 큰 차이는?프로세스는 독립된 메모리 공간을 가지고, 스레드는 같은 프로세스의 메모리 공간을 공유합니다. 이 차이가 통신 비용, 안정성, 동기화 필요성에 영향을 줍니다.
  • 스레드 사용 시 조심할 점은?공유 데이터를 여러 스레드가 동시에 접근할 때 race condition, deadlock, visibility 문제가 생길 수 있습니다. 공유 상태를 줄이고 필요한 곳에만 동기화를 적용해야 합니다.
