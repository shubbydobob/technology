CORS란 무엇인가?

CORS(Cross-Origin Resource Sharing)는 브라우저가 다른 Origin으로 API를 요청할 때 서버가 허용했는지 확인하는 정책입니다. Origin은 프로토콜, 도메인, 포트 조합입니다.

간단한 예시

프론트: http://localhost:5173
API:    http://localhost:8080

프로토콜/호스트/포트 조합이 다르면 Cross-Origin 요청입니다.

주요 헤더

헤더설명
Access-Control-Allow-Origin허용할 출처입니다.
Access-Control-Allow-Methods허용할 HTTP Method입니다.
Access-Control-Allow-Headers허용할 요청 헤더입니다.
Access-Control-Allow-Credentials쿠키 포함 요청 허용 여부입니다.

기술 포인트

  • CORS는 서버 문제인가 브라우저 문제인가?정책을 강제하는 주체는 브라우저이고, 허용 여부를 응답 헤더로 알려주는 쪽은 서버입니다.
  • Preflight란?실제 요청 전에 OPTIONS 요청으로 서버가 해당 요청을 허용하는지 확인하는 과정입니다.
