Operating System

리눅스란 무엇인가?

리눅스는 서버, 클라우드, 임베디드, 개발 환경에서 널리 쓰이는 오픈소스 운영체제 계열입니다.

운영체제

리눅스란 무엇인가?

리눅스는 Linux Kernel을 중심으로 여러 시스템 도구, 라이브러리, 패키지 관리 체계가 결합된 운영체제 계열입니다. 엄밀히 말하면 Linux는 커널이고, Ubuntu, Debian, Rocky Linux 같은 배포판이 사용자가 설치해 쓰는 완성된 운영체제 형태입니다.

핵심 개념

항목설명
커널CPU, 메모리, 디스크, 네트워크 같은 하드웨어 자원을 관리하는 핵심 계층입니다.
셸사용자가 명령어로 운영체제와 상호작용하는 인터페이스입니다. 대표적으로 bash, zsh가 있습니다.
파일 시스템/ 루트를 기준으로 디렉터리가 계층적으로 구성됩니다.
프로세스실행 중인 프로그램 단위이며, 리눅스는 여러 프로세스를 동시에 관리합니다.
권한소유자, 그룹, 기타 사용자 기준으로 읽기, 쓰기, 실행 권한을 제어합니다.

왜 서버에서 많이 쓰는가?

  • 안정성장시간 실행되는 서버 환경에서 안정적으로 동작하도록 설계되어 있습니다.
  • 자동화셸 스크립트, cron, systemd 등을 통해 운영 작업을 자동화하기 쉽습니다.
  • 패키지 생태계웹 서버, DB, 런타임, 모니터링 도구를 패키지 관리자로 설치하고 관리할 수 있습니다.
  • 클라우드 친화성AWS EC2, 컨테이너, Kubernetes 같은 인프라 환경에서 표준처럼 사용됩니다.

대표 배포판

배포판특징
Ubuntu문서와 커뮤니티가 풍부해 개발 서버와 입문 환경에서 많이 사용됩니다.
Debian안정성을 중시하는 배포판이며 Ubuntu의 기반이기도 합니다.
Rocky LinuxRHEL 계열 호환 배포판으로 기업 서버 환경에서 자주 사용됩니다.
Amazon LinuxAWS 환경에 맞춰 제공되는 서버용 배포판입니다.

면접 포인트

  • Linux와 Linux 배포판은 같은가?Linux는 커널을 가리키는 말이고, 배포판은 커널과 도구를 묶은 운영체제 패키지입니다.
  • 리눅스 권한의 기본 단위는?소유자, 그룹, 기타 사용자에게 읽기, 쓰기, 실행 권한을 부여합니다.
  • 서버에서 리눅스를 많이 쓰는 이유는?안정성, 자동화, 패키지 생태계, 클라우드 인프라와의 높은 호환성 때문입니다.
