백엔드

REST API란 무엇인가?

REST API는 서버의 데이터를 자원(Resource)으로 보고, URI와 HTTP Method를 조합해 자원에 대한 행위를 표현하는 API 설계 스타일입니다.

간단한 예시

GET /users        사용자 목록 조회
GET /users/1      사용자 상세 조회
POST /users       사용자 생성
PUT /users/1      사용자 전체 수정
DELETE /users/1   사용자 삭제

설계 기준

기준설명
URI동사보다 명사 중심으로 자원을 표현합니다.
MethodGET, POST, PUT, PATCH, DELETE로 행위를 표현합니다.
Status Code요청 결과를 HTTP 상태 코드로 명확히 전달합니다.
Stateless각 요청은 필요한 인증 정보와 요청 정보를 스스로 포함해야 합니다.

기술 포인트

  • REST API에서 URI는 무엇을 표현하나?행위가 아니라 자원을 표현합니다. 행위는 HTTP Method가 담당합니다.
  • PUT과 PATCH 차이는?PUT은 전체 교체, PATCH는 일부 수정 의미로 주로 사용합니다.
