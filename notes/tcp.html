Transport Layer

TCP란 무엇인가?

TCP는 데이터를 순서대로, 손실 없이 전달하기 위해 연결과 신뢰성을 관리하는 전송 계층 프로토콜입니다.

네트워크

TCP란 무엇인가?

TCP는 Transmission Control Protocol의 약자로, IP 위에서 동작하는 전송 계층 프로토콜입니다. HTTP, HTTPS, SSH, WebSocket처럼 데이터가 정확한 순서로 도착해야 하는 서비스에서 널리 사용됩니다.

핵심 특징

특징설명
연결 지향데이터를 보내기 전에 클라이언트와 서버가 연결을 수립합니다.
신뢰성 보장손실된 데이터는 재전송하고, 수신 확인을 통해 전달 여부를 확인합니다.
순서 보장패킷이 다른 순서로 도착해도 원래 순서대로 재조립합니다.
흐름 제어수신자가 처리할 수 있는 양에 맞춰 전송 속도를 조절합니다.
혼잡 제어네트워크가 혼잡할 때 전송량을 줄여 전체 네트워크 부담을 낮춥니다.

3-way handshake

TCP는 데이터를 보내기 전에 연결을 수립합니다. 대표적인 연결 수립 과정은 3-way handshake입니다.

단계방향의미
1Client -> ServerSYN: 연결을 시작하고 싶다는 요청
2Server -> ClientSYN + ACK: 요청을 받았고 서버도 연결 준비가 됐다는 응답
3Client -> ServerACK: 서버 응답을 확인했고 연결을 확정

TCP와 UDP 차이

구분TCPUDP
연결연결 지향비연결형
신뢰성재전송, 순서 보장보장하지 않음
속도상대적으로 느릴 수 있음상대적으로 빠름
사용 예HTTP, HTTPS, SSH, WebSocketDNS, 실시간 스트리밍, 게임 일부 트래픽

면접 포인트

  • TCP가 신뢰성을 보장한다는 의미는?데이터 손실을 감지하고 재전송하며, 순서를 맞춰 애플리케이션에 전달한다는 의미입니다.
  • 3-way handshake는 왜 필요한가?양쪽이 송수신 가능한 상태인지 확인하고 연결 상태를 동기화하기 위해 필요합니다.
  • TCP가 항상 좋은 선택인가?아닙니다. 지연보다 속도가 중요한 실시간 트래픽에서는 UDP가 더 적합할 수 있습니다.
