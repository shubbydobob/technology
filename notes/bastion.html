배스천이란 무엇인가?

외부에서 내부 서버에 직접 접근하지 않도록 중간 관문 역할을 하는 보안 접속용 서버입니다.

인프라

배스천이란 무엇인가?

배스천 호스트(Bastion Host)는 외부 네트워크에서 private 서버로 접근할 때 거치는 보안 관문 서버입니다. 운영 서버나 DB 서버를 인터넷에 직접 노출하지 않고, 허용된 사용자만 배스천을 통해 내부망으로 접속하게 만드는 구조에서 사용합니다.

간단한 예시

AWS에서 WAS나 DB를 private subnet에 두면 외부에서 직접 SSH 접속이 불가능합니다. 이때 public subnet에 배스천 서버를 두고, 관리자는 먼저 배스천에 접속한 뒤 내부 서버로 다시 접속합니다.

Admin PC
  -> Bastion Host(public subnet)
    -> App Server(private subnet)
    -> DB Server(private subnet)

왜 사용하는가?

목적설명
내부 서버 보호WAS, DB 같은 중요 서버를 인터넷에 직접 노출하지 않습니다.
접근 통제관리자 IP, SSH Key, MFA, 계정 권한을 기준으로 접속을 제한할 수 있습니다.
감사 추적누가 언제 어떤 서버에 접근했는지 로그를 남기기 쉽습니다.
운영 경계 분리외부 접근 지점을 배스천 하나로 모아 보안 정책을 집중 관리할 수 있습니다.

일반적인 접속 흐름

  • 1. 관리자 접속관리자는 허용된 IP와 SSH Key로 배스천 서버에 접속합니다.
  • 2. 내부 서버 이동배스천에서 private IP를 가진 WAS나 DB 서버로 SSH 접속합니다.
  • 3. 작업 수행로그 확인, 배포 확인, 설정 점검 같은 운영 작업을 수행합니다.
  • 4. 로그 기록접속 이력과 명령 기록을 남겨 장애 분석이나 보안 감사에 활용합니다.

보안 설정 포인트

항목권장 방향
Security Group배스천 SSH 포트는 관리자 고정 IP에서만 허용합니다.

권한필요한 서버와 명령에만 접근할 수 있도록 최소 권한을 적용합니다.
대체 서비스AWS Systems Manager Session Manager처럼 키 없는 접속 방식도 검토할 수 있습니다.

주의할 점

배스천은 내부망으로 들어가는 관문이므로 뚫리면 피해가 커질 수 있습니다. 그래서 SSH 포트 전체 공개, 공용 계정 사용, 개인키 공유, 로그 미수집 같은 운영 방식은 피해야 합니다.

기술 포인트

  • 배스천의 핵심 역할은?내부 서버를 외부에 직접 노출하지 않고, 관리자가 통제된 경로로만 접근하게 만드는 보안 관문입니다.
  • 배스천과 일반 서버의 차이는?배스천은 애플리케이션 서비스를 제공하기보다 내부 서버 접근을 통제하고 기록하는 운영 접속용 서버입니다.
  • 배스천을 안전하게 운영하려면?접속 IP 제한, 사용자별 키 관리, MFA, 로그 수집, 최소 권한 적용이 필요합니다. 가능하면 Session Manager 같은 관리형 접속 방식도 함께 검토합니다.
