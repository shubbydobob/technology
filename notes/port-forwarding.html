외부에서 들어온 특정 포트 요청을 내부 네트워크의 특정 장비와 포트로 전달하는 네트워크 설정입니다.

네트워크

포트 포워딩이란 무엇인가?

포트 포워딩은 공유기나 방화벽 같은 네트워크 장비가 받은 요청을 내부 서버의 특정 포트로 넘겨주는 설정입니다. NAT 환경에서 외부 사용자가 내부 서버에 접근해야 할 때 자주 사용합니다.

예시

공유기의 공인 IP로 들어온 80번 포트 요청을 내부 서버 192.168.0.10:8080으로 전달하도록 설정할 수 있습니다.

외부 요청전달 대상의미
공인IP:80192.168.0.10:8080웹 요청을 내부 애플리케이션 서버로 전달
공인IP:2222192.168.0.20:22외부 SSH 접속을 내부 서버로 전달

사용 사례

  • 개발 서버 공개내부 PC에서 실행 중인 웹 서버를 외부에서 테스트해야 할 때 사용합니다.
  • 홈 서버 운영NAS, 게임 서버, 웹 서버를 공유기 뒤에서 외부에 공개할 때 사용합니다.
  • 포트 변환외부 포트와 내부 서비스 포트를 다르게 매핑할 수 있습니다.

주의할 점

포트 포워딩은 내부 서비스를 외부에 노출합니다. 따라서 인증, 방화벽, 접근 IP 제한, 서비스 업데이트 상태를 함께 점검해야 합니다.

면접 포인트

  • NAT와 어떤 관계가 있는가?NAT 뒤의 내부 사설 IP 서버로 외부 요청을 전달하기 위해 포트 포워딩을 사용합니다.
  • 포트를 열면 바로 접속 가능한가?아닙니다. OS 방화벽, 애플리케이션 바인딩 주소, ISP 정책도 함께 확인해야 합니다.
  • 보안 위험은?공개된 포트가 공격 대상이 될 수 있으므로 최소한의 포트만 열고 접근 제어를 적용해야 합니다.
