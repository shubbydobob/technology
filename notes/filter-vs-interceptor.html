백엔드

필터와 인터셉터 차이

Filter는 서블릿 컨테이너 영역에서 동작하고, Interceptor는 Spring MVC 영역에서 Controller 호출 전후에 동작합니다.

실행 흐름

Client -> Filter -> DispatcherServlet -> Interceptor -> Controller

비교

구분FilterInterceptor
소속Servlet ContainerSpring MVC
위치DispatcherServlet 앞Controller 앞/뒤
대상모든 요청 수준HandlerMapping된 요청
예시CORS, 인코딩, 보안 필터권한 체크, 로깅, 공통 모델 처리

기술 포인트

  • 인증은 어디에 둘까?보안 프레임워크는 보통 Filter 체인에서 처리하고, 화면/권한성 부가 검사는 Interceptor로 처리할 수 있습니다.
  • Interceptor가 정적 파일에도 적용되나?Spring MVC HandlerMapping 대상이 아니면 적용되지 않을 수 있습니다.
