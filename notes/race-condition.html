Race Condition이란 무엇인가?

여러 실행 흐름이 공유 자원에 동시에 접근하면서 실행 순서에 따라 결과가 달라지는 문제입니다.

백엔드

Race Condition이란 무엇인가?

Race Condition은 여러 스레드나 요청이 같은 데이터를 동시에 수정할 때 실행 순서에 따라 의도하지 않은 결과가 생기는 상황입니다.

간단한 예시

count = 0
Thread A: count 읽기 -> 0
Thread B: count 읽기 -> 0
Thread A: count = 1 저장
Thread B: count = 1 저장

기대값은 2지만 결과는 1

해결 방식

방식설명
synchronized한 번에 하나의 스레드만 임계 구역에 들어가게 합니다.
Lock락 획득과 해제를 명시적으로 제어합니다.
Atomic원자적 연산을 제공하는 타입을 사용합니다.
DB Lock데이터베이스 레벨에서 동시 수정을 제어합니다.

기술 포인트

  • 임계 구역이란?공유 자원에 접근해 동시에 실행되면 문제가 생길 수 있는 코드 영역입니다.
  • 동기화의 비용은?정합성은 높아지지만 대기 시간이 늘고 데드락 위험이 생길 수 있습니다.
