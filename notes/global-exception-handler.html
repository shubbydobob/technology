전역 예외 핸들러란 무엇인가?

여러 컨트롤러에서 발생하는 예외를 한 곳에서 처리해 일관된 에러 응답을 만드는 방식입니다.

백엔드

전역 예외 핸들러란 무엇인가?

전역 예외 핸들러는 API에서 발생하는 예외를 공통으로 잡아 상태 코드와 에러 메시지 형식을 통일하는 구조입니다. Spring에서는 @RestControllerAdvice와 @ExceptionHandler를 많이 사용합니다.

간단한 예시

@RestControllerAdvice
class GlobalExceptionHandler {
  @ExceptionHandler(NotFoundException.class)
  ResponseEntity handle(NotFoundException e) {
    return ResponseEntity.status(404).body("not found");
  }
}

왜 필요한가?

이유설명
일관성API 에러 응답 형식을 통일합니다.
중복 제거각 컨트롤러의 반복 try-catch를 줄입니다.
보안내부 예외 상세를 클라이언트에 노출하지 않습니다.
관측성로그와 추적 ID를 공통으로 남길 수 있습니다.

기술 포인트

  • 모든 예외를 500으로 보내도 되나?아닙니다. 검증 실패, 인증 실패, 권한 실패, 리소스 없음 등은 의미에 맞는 상태 코드를 써야 합니다.
  • 좋은 에러 응답은?상태 코드, 에러 코드, 메시지, 추적 ID처럼 클라이언트와 운영자가 이해할 정보를 포함해야 합니다.
