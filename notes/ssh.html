네트워크

SSH란 무엇인가?

SSH는 Secure Shell의 약자로, 네트워크를 통해 원격 서버에 안전하게 접속하기 위한 프로토콜입니다. 기본 포트는 22번이며, 서버 관리, 파일 전송, 터널링에 자주 사용됩니다.

무엇을 안전하게 하는가?

기능설명
암호화클라이언트와 서버 사이의 데이터를 암호화해 중간에서 내용을 읽기 어렵게 합니다.
서버 검증호스트 키를 통해 접속 대상 서버가 이전에 접속한 서버와 같은지 확인할 수 있습니다.
사용자 인증비밀번호 또는 공개키 기반 인증으로 사용자를 확인합니다.
명령 실행원격 서버의 셸에서 명령어를 실행할 수 있습니다.

인증 방식

  • 비밀번호 인증계정 비밀번호로 접속합니다. 설정은 쉽지만 무차별 대입 공격에 취약할 수 있습니다.
  • 공개키 인증개인키는 클라이언트에 보관하고 공개키는 서버에 등록해 인증합니다. 운영 서버에서는 이 방식을 선호합니다.
  • MFA 연동보안이 중요한 환경에서는 추가 인증을 함께 적용할 수 있습니다.

자주 쓰는 명령 예시

명령의미
ssh user@host원격 서버에 SSH로 접속합니다.
ssh -i key.pem user@host특정 개인키를 사용해 접속합니다.
scp file user@host:/pathSSH 기반으로 파일을 복사합니다.

면접 포인트

  • SSH의 기본 포트는?기본값은 TCP 22번 포트입니다.
  • 공개키 인증은 어떻게 동작하는가?서버는 등록된 공개키를 기준으로 클라이언트가 대응되는 개인키를 가지고 있는지 검증합니다.
  • SSH와 Telnet의 차이는?Telnet은 평문 통신이고, SSH는 암호화된 통신을 제공합니다.
