백엔드

ORM이란 무엇인가?

ORM(Object Relational Mapping)은 객체와 관계형 데이터베이스 테이블을 연결해주는 기술입니다. 개발자는 SQL을 직접 많이 작성하지 않고 객체를 저장, 조회, 수정, 삭제하는 방식으로 데이터베이스를 다룰 수 있습니다.

간단한 예시

사용자 정보를 저장할 때 SQL을 직접 작성하는 대신 User 객체를 만들고 저장 메서드를 호출하면 ORM이 적절한 SQL을 생성해 데이터베이스에 반영합니다.

// 객체
User user = new User("kim", "kim@example.com");
userRepository.save(user);

// ORM이 내부적으로 생성할 수 있는 SQL
INSERT INTO users (name, email) VALUES ('kim', 'kim@example.com');

왜 사용하는가?

이유설명
생산성반복적인 CRUD SQL 작성을 줄일 수 있습니다.
객체 중심 개발비즈니스 로직을 테이블보다 도메인 객체 중심으로 표현할 수 있습니다.
유지보수매핑 규칙을 기준으로 데이터 접근 코드를 일관되게 관리할 수 있습니다.
DB 독립성일부 SQL 차이를 ORM이 추상화해 DB 변경 부담을 줄일 수 있습니다.

대표 개념

  • EntityDB 테이블과 매핑되는 객체입니다. 예를 들어 User 클래스가 users 테이블과 연결될 수 있습니다.
  • RepositoryEntity를 저장하거나 조회하는 데이터 접근 계층입니다.
  • Mapping객체의 필드와 테이블의 컬럼을 연결하는 규칙입니다.
  • Persistence ContextJPA에서 Entity를 관리하는 1차 캐시이자 변경 감지의 기준이 되는 영역입니다.

주의할 점

주의점설명
N+1 문제연관 데이터를 조회할 때 예상보다 많은 SQL이 실행될 수 있습니다.
복잡한 쿼리통계, 리포트, 복잡한 조인 쿼리는 직접 SQL이나 QueryDSL이 더 적합할 수 있습니다.
성능 이해ORM을 써도 실제 실행 SQL과 인덱스, 트랜잭션 범위를 이해해야 합니다.
매핑 설계객체 관계와 테이블 관계를 무리하게 맞추면 코드와 쿼리 모두 복잡해질 수 있습니다.

구분ORMSQL Mapper

기술 포인트

  • ORM은 SQL을 몰라도 된다는 뜻인가?아닙니다. ORM이 SQL 생성을 도와주지만 성능 문제를 잡으려면 실제 실행 SQL, 조인, 인덱스, 트랜잭션을 이해해야 합니다.
  • JPA와 Hibernate의 관계는?JPA는 자바 ORM 표준 인터페이스이고 Hibernate는 그 구현체 중 하나입니다. Spring Data JPA는 JPA 사용을 더 편하게 해주는 추상화입니다.
  • N+1 문제란?목록 1번 조회 후 각 row의 연관 데이터를 다시 N번 조회해 SQL이 과도하게 실행되는 문제입니다. fetch join, EntityGraph, batch size 같은 방식으로 줄일 수 있습니다.
