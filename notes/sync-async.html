백엔드

동기와 비동기란 무엇인가?

동기는 호출한 작업의 결과를 기다린 뒤 다음 흐름으로 진행하는 방식입니다. 비동기는 작업을 맡겨두고 결과가 준비되면 콜백, 이벤트, Future 같은 방식으로 처리합니다.

간단한 예시

동기: 주문 저장 -> 결제 처리 완료 대기 -> 응답
비동기: 주문 저장 -> 메시지 발행 -> 바로 응답 -> 결제는 나중에 처리

비교

구분설명
동기흐름이 단순하고 결과를 바로 알 수 있지만 대기 시간이 길 수 있습니다.
비동기응답성을 높일 수 있지만 상태 추적과 실패 처리가 복잡해집니다.
블로킹스레드가 작업 완료까지 멈춰 있습니다.
논블로킹대기 중에도 스레드가 다른 작업을 처리할 수 있습니다.

기술 포인트

  • 비동기가 항상 좋은가?아닙니다. 흐름 추적, 재시도, 순서 보장, 장애 처리 비용이 증가할 수 있습니다.
  • 동기/비동기와 블로킹/논블로킹은 같은가?다릅니다. 동기/비동기는 결과 처리 방식, 블로킹/논블로킹은 대기 중 제어권 관점입니다.
