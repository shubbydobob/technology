Vue3의 컴포지션 API란 무엇인가?

Composition API는 컴포넌트의 상태와 로직을 함수 기반으로 구성해 재사용성과 유지보수성을 높이는 Vue3의 API입니다.

프론트엔드

Vue3의 컴포지션 API란 무엇인가?

Vue3의 Composition API는 setup, ref, reactive, computed, watch 등을 사용해 컴포넌트 로직을 작성하는 방식입니다. Options API처럼 data, methods, computed 옵션별로 나누기보다, 관련 기능을 한 곳에 묶어 관리할 수 있습니다.

간단한 예시

<script setup>
import { ref, computed } from 'vue'

const count = ref(0)
const doubled = computed(() => count.value * 2)

function increase() {
  count.value++
}
</script>

<template>
  <button @click="increase">{{ count }} / {{ doubled }}</button>
</template>

ref는 반응형 값을 만들고, computed는 상태를 기반으로 계산된 값을 만듭니다. 템플릿에서는 .value 없이 사용할 수 있습니다.

왜 사용하는가?

  • 로직 재사용검색, 페이지네이션, 폼 처리 같은 로직을 composable 함수로 분리할 수 있습니다.
  • 관심사 분리기능별 상태와 함수를 가까이 배치할 수 있어 큰 컴포넌트에서 흐름을 파악하기 쉽습니다.
  • 타입 추론TypeScript와 함께 사용할 때 Options API보다 타입 추론이 자연스러운 경우가 많습니다.

면접 포인트

  • Composition API의 핵심 장점은?관련 로직을 기능 단위로 묶고 재사용하기 쉽다는 점입니다.
  • ref와 reactive 차이는?ref는 단일 값이나 객체를 감싸고 .value로 접근하며, reactive는 객체 자체를 반응형으로 만듭니다.
  • Options API는 더 이상 못 쓰는가?아닙니다. Vue3에서도 Options API는 사용할 수 있으며, 프로젝트 규모와 팀 규칙에 맞춰 선택합니다.
