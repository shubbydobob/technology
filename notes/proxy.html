프록시란 무엇인가?

클라이언트와 서버 사이에서 요청과 응답을 대신 전달하거나 제어하는 중간 서버입니다.

네트워크

프록시란 무엇인가?

프록시(Proxy)는 클라이언트와 서버 사이에 위치해 요청을 대신 보내거나 응답을 대신 받아 전달하는 중간 서버입니다. 접근 제어, 캐싱, 로깅, 보안, 내부 서버 보호, 트래픽 라우팅 같은 목적으로 사용됩니다.

간단한 예시

회사 내부 PC가 인터넷에 직접 나가지 않고 회사 프록시 서버를 거쳐 외부 사이트에 접속할 수 있습니다. 반대로 사용자가 서비스에 접속할 때 Nginx가 앞단에서 요청을 받아 내부 WAS로 전달하는 것도 프록시 구조입니다.

Forward Proxy:
Client -> Proxy -> Internet Server

Reverse Proxy:
Client -> Proxy(Nginx) -> Internal WAS

주요 역할

  • 접근 제어허용된 사용자나 네트워크만 특정 서버에 접근하게 제한할 수 있습니다.
  • 캐싱자주 요청되는 응답을 저장해 서버 부하와 응답 시간을 줄일 수 있습니다.
  • 보안내부 서버 주소를 숨기고 요청을 검사해 공격이나 비정상 요청을 차단할 수 있습니다.
  • 라우팅도메인, 경로, 헤더 기준으로 요청을 여러 내부 서비스에 나누어 전달할 수 있습니다.

Nginx와의 관계

Nginx는 리버스 프록시로 자주 사용됩니다. 예를 들어 /api 요청은 Spring Boot 서버로 보내고, 정적 파일 요청은 Nginx가 직접 응답하게 구성할 수 있습니다.

location /api/ {
  proxy_pass http://127.0.0.1:8080/;
}

기술 포인트

  • 프록시의 핵심은?클라이언트와 서버 사이에서 통신을 대신 처리하는 중간 계층이라는 점입니다. 단순 전달뿐 아니라 제어, 기록, 캐싱, 보안 정책을 적용할 수 있습니다.
  • 포워드 프록시와 리버스 프록시의 차이는?포워드 프록시는 클라이언트 보호와 외부 접속 제어가 중심이고, 리버스 프록시는 서버 보호와 내부 서비스 라우팅이 중심입니다.
  • 리버스 프록시를 쓰는 이유는?내부 서버를 직접 노출하지 않고 HTTPS 처리, 로드밸런싱, 경로 기반 라우팅, 공통 보안 정책을 앞단에서 처리하기 위해 사용합니다.
