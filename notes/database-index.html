인덱스란 무엇인가?

데이터베이스에서 원하는 데이터를 빠르게 찾기 위해 별도로 관리하는 검색용 자료구조입니다.

백엔드

인덱스란 무엇인가?

인덱스는 테이블의 특정 컬럼을 기준으로 검색 속도를 높이기 위한 구조입니다. 책의 목차처럼 원하는 위치를 빠르게 찾게 해주지만, 쓰기 작업에는 추가 비용이 발생합니다.

간단한 예시

CREATE INDEX idx_users_email ON users(email);
SELECT * FROM users WHERE email = 'kim@example.com';

장단점

구분설명
장점WHERE, JOIN, ORDER BY 성능을 개선할 수 있습니다.
단점INSERT, UPDATE, DELETE 시 인덱스 갱신 비용이 듭니다.
주의선택도가 낮은 컬럼에는 효과가 작을 수 있습니다.
확인EXPLAIN으로 실제 실행 계획을 확인해야 합니다.

기술 포인트

  • 인덱스는 항상 좋은가?아닙니다. 조회는 빨라질 수 있지만 저장 공간과 쓰기 비용이 증가합니다.
  • 복합 인덱스에서 중요한 점은?컬럼 순서가 중요합니다. 조건과 정렬 패턴에 맞춰 설계해야 합니다.
