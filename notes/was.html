WAS란 무엇인가?

WAS(Web Application Server)는 웹 애플리케이션의 비즈니스 로직을 실행하고, 동적인 응답을 생성해주는 서버입니다.

핵심 개념

항목설명
정식 의미Web Application Server
주 역할동적 웹 애플리케이션 실행
대표 기능비즈니스 로직 처리, DB 연동, 세션/트랜잭션 관리
대표 예시Tomcat, JBoss/WildFly, WebLogic, WebSphere
자주 쓰는 환경Java Servlet/JSP, Spring 기반 애플리케이션

Web Server와 WAS 차이

구분Web ServerWAS
주요 역할정적 리소스 제공동적 로직 실행
처리 대상HTML, CSS, JS, 이미지API, DB 조회, 인증, 비즈니스 로직
대표 예시Nginx, Apache HTTP ServerTomcat, JBoss, WebLogic
강점빠른 정적 파일 처리, 리버스 프록시, 로드밸런싱애플리케이션 실행, 트랜잭션, 세션 처리

요청 처리 흐름

flowchart LR A[Client Browser] --> B[Web Server] B --> C{정적 리소스인가?} C -- 예 --> D[HTML/CSS/JS/Image 응답] C -- 아니오 --> E[WAS로 요청 전달] E --> F[Application Logic 실행] F --> G[DB 조회/외부 API 호출] G --> H[동적 응답 생성] H --> B B --> A

WAS가 필요한 이유

이유설명
동적 응답 생성사용자, 요청 파라미터, DB 상태에 따라 다른 응답을 만듭니다.
비즈니스 로직 실행주문, 결제, 로그인, 권한 체크 같은 핵심 로직을 처리합니다.
DB 연동데이터 조회/저장/수정/삭제 작업을 수행합니다.
트랜잭션 관리여러 DB 작업을 하나의 논리적 작업으로 묶어 일관성을 보장합니다.
세션 관리로그인 상태 등 사용자별 상태를 관리할 수 있습니다.

면접 포인트

  • WAS와 Web Server의 차이는?Web Server는 정적 리소스와 프록시에 강하고, WAS는 애플리케이션 로직 실행에 초점이 있습니다.
  • Tomcat은 Web Server인가 WAS인가?Servlet 컨테이너이자 WAS로 볼 수 있습니다. 다만 정적 파일 처리도 가능합니다.
  • 왜 Nginx와 WAS를 같이 쓰는가?Nginx가 정적 파일, TLS, 로드밸런싱, 리버스 프록시를 맡고 WAS는 애플리케이션 로직에 집중하게 하기 위해서입니다.
