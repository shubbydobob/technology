## 포트 포워딩이란 무엇인가?

포트 포워딩은 외부에서 들어온 네트워크 요청을 내부 네트워크의 특정 장비와 포트로 전달하는 설정입니다.

예를 들어 집이나 회사 공유기의 공인 IP로 `80`번 포트 요청이 들어왔을 때, 내부 서버 `192.168.0.10:8080`으로 보내도록 설정할 수 있습니다. 외부 사용자는 공유기 주소로 접속하지만, 실제 요청은 내부 서버의 애플리케이션까지 전달됩니다.

주요 사용 사례:

- 외부에서 내부 개발 서버에 접속해야 할 때
- 공유기 뒤에 있는 웹 서버, 게임 서버, NAS를 외부에 공개할 때
- 특정 포트 요청을 다른 내부 포트로 매핑해야 할 때

주의할 점:

- 내부 서비스가 외부에 노출되므로 방화벽, 인증, 접근 제한을 함께 고려해야 합니다.
- 공인 IP, 공유기 설정, 내부 서버 IP 고정 여부에 영향을 받습니다.
- 포트를 열었다고 서비스가 항상 접속 가능한 것은 아니며, OS 방화벽과 애플리케이션 바인딩 주소도 확인해야 합니다.

## 터널링은 무엇인가?

터널링은 한 네트워크 프로토콜의 데이터를 다른 네트워크 연결 안에 감싸서 전달하는 방식입니다. 직접 접근하기 어려운 네트워크나 서버에 중간 통로를 만들어 접속할 때 자주 사용합니다.

대표적인 예시는 SSH 터널링입니다. 로컬 PC에서 접근할 수 없는 내부 DB가 있을 때, SSH로 접근 가능한 서버를 경유해 로컬 포트와 내부 DB 포트를 연결할 수 있습니다.

예시 흐름:

```text
로컬 PC:3306 -> SSH 서버 -> 내부 DB:3306
```

이 경우 로컬에서는 `localhost:3306`으로 접속하지만, 실제 트래픽은 SSH 연결을 통해 내부 DB까지 전달됩니다.

주요 사용 사례:

- 사설망에 있는 DB나 관리자 페이지에 안전하게 접근할 때
- 방화벽 때문에 직접 접근할 수 없는 서비스를 우회 접속할 때
- 로컬 개발 환경과 원격 서버 환경을 임시로 연결할 때
- ngrok, Cloudflare Tunnel처럼 로컬 서버를 임시 공개 URL로 노출할 때

포트 포워딩과의 차이:

| 구분 | 포트 포워딩 | 터널링 |
|---|---|---|
| 목적 | 외부 요청을 내부 특정 포트로 전달 | 별도 연결 통로 안에 트래픽을 실어 전달 |
| 설정 위치 | 주로 공유기, 방화벽, NAT 장비 | SSH, VPN, 터널링 도구, 프록시 |
| 보안 성격 | 포트를 직접 노출할 수 있음 | 암호화된 통로를 쓰는 경우가 많음 |
| 사용 예 | 공유기 `80`번 포트를 내부 서버 `8080`으로 연결 | 로컬 `3306`을 SSH 경유로 내부 DB에 연결 |
